Day 4 Journal: Design Systems, Network Battles, and Headless UI
What I Built
Today was a massive milestone for Naija Commerce: establishing a proper design system using Tailwind v4 and shadcn/ui. I didn't just add a few buttons; I built a complete, living style guide on a dedicated /design route. This page showcases Light and Dark themes side-by-side using CSS variables for design tokens, beautifully integrating the Nigerian Naira (₦) for our local context. I integrated ten core components, including complex ones like Dialogs, Data Tables, and Forms. I also learned the magic behind the cn() utility—how it uses clsx for conditional logic and tailwind-merge to resolve styling conflicts, giving me total control over my UI without breaking the default layouts.

What Broke
The terminal almost defeated me today. When I tried to run the standard pnpm dlx shadcn@latest add command, my internet connection kept dropping, throwing relentless ECONNRESET errors. The CLI was trying to make too many network requests at once and completely froze. But I pivoted. Instead of fighting the CLI, I learned that shadcn isn't a locked node_modules library—it's just code you own. I bypassed the terminal, manually created the component files, pasted the bleeding-edge React 19 code straight from the docs, and used a single direct pnpm command to install the Radix UI engines. A frustrating blocker turned into a huge "aha" moment.

What I Don’t Yet Understand
While the "headless" UI concept clicks in theory—Radix handles the invisible accessibility "bones" while shadcn provides the Tailwind "skin"—I'm still wrapping my head around the deeper mechanics of React 19. Seeing React.ComponentProps<"div"> and the new data-slot attributes replace the old forwardRef pattern makes the code look cleaner, but I don't fully grasp how the CSS styling links up to those data-slots under the hood just yet. Additionally, the setup for the Form component (combining react-hook-form, zod, and context providers) looks incredibly powerful but very complex. I know it works, but building a multi-step checkout form using this architecture is definitely going to require some deep studying.
