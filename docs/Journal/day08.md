Day 8 Project Journal: Securing Naija Commerce

What I Built Today
Today was a massive architectural leap forward: I completely wired up the authentication and security layer for Naija Commerce. I successfully integrated Better-Auth into the Next.js 15 App Router, moving away from relying on managed services to deeply understand session-based auth. I updated my Drizzle ORM schema to include tables for users, sessions, and accounts, pushed the changes to my Supabase Postgres database, and built a dynamic frontend login UI to handle both passwordless Magic Links and Google OAuth. The crown jewel of the day was engineering the Next.js Edge Middleware. I built a customized "bouncer" that protects the /account and /checkout routes, and strictly enforces Role-Based Access Control (RBAC) by completely locking customer accounts out of the /dashboard.

What Broke
TypeScript and React structure were my biggest nemeses today. I spent time fighting a "Cannot find module" error, which turned out to be a simple incomplete pnpm installation rather than bad code. In the UI, I fell into a few classic React traps: trying to call hooks like useState outside of my main component, and accidentally triggering sign-out and magic link functions the millisecond the page loaded because I didn't wrap my onClick handlers in arrow functions. Building the middleware was also a heavy logic puzzle; I initially nested my if statements incorrectly, which would have caused the app to crash by checking properties on a null session.

What I Don't Yet Understand
While the middleware bouncer is fully operational, the "Edge" execution environment in Next.js is still a bit fuzzy to me. I had to use a specialized fetch tool (@better-fetch/fetch) to manually pass cookies and verify the session in the middleware, and I don't fully grasp why a standard API call doesn't work there yet. Additionally, TypeScript's strictness around my custom role column clashing with Better-Auth's default Session type was confusing. I managed to bypass it cleanly using optional chaining (session?.user?.role), but I still need to learn how to formally extend library types in TypeScript so it inherently trusts my database schema.
